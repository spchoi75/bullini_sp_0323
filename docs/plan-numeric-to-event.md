# 기능 #5: Numeric→Event θ 임계값 추정

## 핵심 개념
"수치 지표가 특정 값에 도달하면 사건이 트리거되는 관계"를 다룬다.
예: "환율이 1,400원 → 외환위기 경고", "실업률 6% → 경기침체 선언"

### Hard vs Soft Threshold

| | Hard Threshold | Soft Threshold |
|---|---|---|
| **동작** | θ를 넘는 순간 P=0→1 점프 | θ에 가까워질수록 P가 서서히 상승 |
| **수학** | P = I(X ≥ θ) (지시함수) | P = 1 / (1 + exp(-k·(X-θ))) |
| **적용** | 법규/규제 명시 기준 | 시장 심리, 투자자 공포 |
| **현실성** | 낮음 (칼같이 변하지 않음) | **높음** (점진적 전이가 현실적) |
| **파라미터** | θ 하나 | θ (중심점) + k (기울기=민감도) |

### 3경로 파이프라인
1. **규제/공식 임계값 조회** (Haiku): 법적 기준 → Hard, 공식 목표 → Soft
2. **Logistic Regression** (Python): 과거 데이터 → θ=-β₀/β₁, k=β₁
3. **LLM 관행적 추정** (Haiku): 시장 통용 기준 + 민감도(k) 추정
