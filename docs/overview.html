01 전체 시스템 아키텍처

Bullini Causal Map은 2-티어 아키텍처로 구성됩니다. 프론트엔드는 Next.js 기반의 3-패널 레이아웃으로, 뉴스/토픽 입력 패널, React Flow 기반 인과 그래프 시각화, 상세 정보 패널을 제공합니다. 백엔드는 Next.js API Routes가 주요 비즈니스 로직을 처리하며, 통계 연산이 필요한 경우 Python FastAPI 서비스로 위임합니다. 외부 API를 통해 뉴스 검색, LLM 추론, 경제 데이터를 수집합니다.

02 파이프라인 흐름

사용자가 투자 토픽을 입력하거나 뉴스를 검색하면, 4단계 파이프라인을 통해 인과 그래프가 생성되고 시각화됩니다. 각 단계는 독립적으로 실행되며, 이전 단계의 출력이 다음 단계의 입력이 됩니다.

03 인과 그래프 구조 예시

아래는 "한국 반도체 주가지수"를 최종 노드로 하는 3개의 인과 체인 예시입니다. 각 체인은 서로 다른 관점(수요, 공급, 지정학)에서 최종 변수에 영향을 미치는 경로를 나타냅니다. 그래프 상에서 체인들이 최종 노드로 수렴하는 구조를 보여줍니다.

04 4가지 엣지 타입

인과 그래프의 엣지는 연결하는 노드의 성격에 따라 4가지 타입으로 분류됩니다. 각 타입마다 서로 다른 파라미터를 추정하며, 이를 통해 정량적 임팩트 계산이 가능해집니다.

엣지 타입 원인 노드 결과 노드 파라미터 설명
Event → Numeric 이벤트 수치 변수 Δ (delta) 이벤트 발생 시 수치 변수의 예상 변화량
Numeric → Numeric 수치 변수 수치 변수 β, r, p 회귀계수, 상관계수, 유의확률 (통계적 인과 강도)
Event → Event 이벤트 이벤트 P 원인 이벤트가 결과 이벤트를 유발할 전이 확률
Numeric → Event 수치 변수 이벤트 θ (theta) 수치 변수가 임계값을 넘으면 이벤트 트리거

05 기술 스택

프론트엔드, 백엔드, 외부 서비스, 개발 도구로 구성된 기술 스택입니다. Next.js를 중심으로 프론트엔드와 API 레이어를 통합하고, 통계 연산은 Python FastAPI에 위임하는 구조를 채택했습니다.
